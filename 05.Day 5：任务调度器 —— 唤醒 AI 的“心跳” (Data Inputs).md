# 🚀 Day 5：任务调度器 —— 唤醒 AI 的“心跳” (Data Inputs)

**今日目标**：深入 `TA-peak-llm-analyzer` 底层目录，解剖 Add-on Builder (AOB) 为我们自动生成的 Python 骨架代码。编写并注入一段“心跳”测试代码，并在 Splunk 搜索栏中成功抓取到它！

在 Day 4 中，我们创建了一个名为 `peak_hunter_task` 的数据输入任务，并设定了 `300` 秒（5 分钟）的执行间隔。这就相当于我们给 AI 设定了一个“闹钟”。今天，我们要让这个闹钟响起来。

### 🛠️ Step 1：潜入底层，寻找灵魂载体 (Python 脚本)

当我们昨天在 AOB 中点击 Save 的那一刻，AOB 已经在后台为你写好了模块化输入 (Modular Input) 的基础 Python 框架。

打开你的 Ubuntu WSL 终端，直接进入我们锁定的插件目录：
```bash
# 1. 进入插件的核心代码目录
cd /opt/splunk/etc/apps/TA-peak-llm-analyzer/bin/

# 2. 查看 AOB 生成的文件
ls -l
```
你会在列表中看到一个名为 `peak_hunter_task.py` 的文件。这就是我们整个 AI 插件跳动的心脏！

---

### 🧠 Step 2：核心代码解剖 (Anatomy of the Code)

我们使用 `nano` 或 `vi` 打开这个文件：
```bash
nano peak_hunter_task.py
```
向下滚动，你会看到两个极其重要的 Python 函数（这是 AOB 的标准规范）：

1. `def validate_input(helper, definition):`
   * **作用**：用来校验用户输入的参数是否合法（比如检查 API Key 是否为空）。我们今天暂时不需要修改它。
2. `def collect_events(helper, ew):`
   * **作用**：**这是绝对的核心！** 每隔 300 秒，Splunk 底层调度器就会自动运行一次这个函数。我们未来所有调用大模型、处理日志的逻辑，都要写在这个函数里。

---

### 💻 Step 3：注入“心跳”测试代码

我们要修改 `collect_events` 函数，让它去读取我们在 Day 4 设置的全局变量（Global Setup），然后生成一条测试日志写入 Splunk。

在 `nano` 编辑器中，找到 `def collect_events(helper, ew):`，将它下方的内容**完全替换/修改**为以下极简代码：

```python
def collect_events(helper, ew):
    """Implement your data collection logic here"""
    
    # 1. 在后台终端打印一条调试日志 (这会记录在 splunkd.log 中)
    helper.log_info("🚀 PEAK AI Hunter 任务被调度器唤醒！开始执行...")

    try:
        # 2. 读取我们在 Day 4 配置的全局变量
        # 注意：使用 AOB 官方提供的 get_global_setting 魔法函数自动解密
        model_name = helper.get_global_setting("model_name")
        
        # 3. 构造我们要写入 Splunk 的业务日志内容
        log_data = f"PEAK AI Hunter 引擎心跳正常。当前挂载的大模型是: {model_name}"
        
        # 4. 创建一个标准的 Splunk 事件对象
        event = helper.new_event(
            source=helper.get_input_type(),
            index=helper.get_output_index(),
            sourcetype=helper.get_sourcetype(),
            data=log_data
        )
        
        # 5. 将事件写入 Splunk 的索引库
        ew.write_event(event)
        
        helper.log_info("✅ 心跳日志已成功写入 Splunk。")
        
    except Exception as e:
        # 极客习惯：永远做好异常捕获
        helper.log_error(f"❌ 运行发生错误: {str(e)}")
```

**保存并退出**：按 `Ctrl + O`，回车确认，然后按 `Ctrl + X` 退出。

### 🛠️ Step 4：在前端激活任务并抓取日志

代码写好了，现在我们要去 Splunk 前端把这个“开关”打开。

1. 在浏览器中打开 Splunk (`http://localhost:8000`)。
2. 在左侧进入你的应用 **PEAK-llm-analyzer**。
3. 点击顶部导航栏的 **Inputs (输入)** 标签页。
4. 点击右上角的 **Create New Input (创建新输入)** -> 选择 **PEAK AI Hunter**。
5. 在弹出的表单中填写基本信息：
   * **Name**: `test_heartbeat`
   * **Interval**: `60` *(为了方便测试，我们覆盖默认的 300 秒，让它每 60 秒跑一次)*
   * **Index**: 选择 `default` 或 `main`。
6. 点击 **Add**。

**🔍 见证奇迹的时刻（搜索验证）：**
任务激活后，耐心等待大概一分钟，让调度器跑完第一轮。
然后点击顶部导航栏的 **Search (搜索)** 标签页，在黑色的搜索框中输入我们 Day 4 定义的 sourcetype：

```spl
sourcetype="peak:llm:execution"
```

调整时间范围为 **Last 15 minutes**，点击搜索。
如果一切顺利，你将在搜索结果中看到一条崭新的绿色日志：
> `PEAK AI Hunter 引擎心跳正常。当前挂载的大模型是: qwen3.5-plus`
