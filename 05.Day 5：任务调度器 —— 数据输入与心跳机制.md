Day 5：任务调度器 —— 建立后台数据输入流与心跳机制 (Data Inputs)

今日目标：理解 Data Input（数据输入）的底层调度原理。我们将顺应 Add-on Builder (AOB) 的强制参数规则配置输入界面，编写 Python 探针代码，并在前端真实启动任务。最终，我们要去 Splunk 的内部索引 _internal 中，亲眼见证代码被系统定时唤醒的运行轨迹。

极客白话：什么是 Data Input 与心跳机制？

对于初学者来说，Data Input（数据输入）听起来像是一个让你填表单的界面。但实际上，它是 Splunk 插件的心脏起搏器！

1. 频率 (Interval) = 心率：当你在界面上设置了 Interval 为 60 秒时，相当于给系统设定了一个定时闹钟。
2. 执行 (Execution) = 心跳：每隔 60 秒，Splunk 的底层调度器就会自动去唤醒并执行一次你写的 Python 脚本，就像心脏跳动一样。
3. 多实例 (Multi-instance)：你可以基于同一个 Python 脚本，在前端创建无数个任务（比如任务 A 每 60 秒查防火墙，任务 B 每 10 分钟查数据库），它们独立跳动，互不干扰。

今天，我们要给这颗心脏装上探针，记录下它的每一次跳动！

Step 1：配置前端 UI (化被动为主动)

AOB 有一个硬性的系统规定：数据输入模块必须至少保留一个自定义参数，否则无法保存并进入下一步。既然我们不能把它删光，那就把它改造成一个有用的探针字段。

详细操作路径：
1. 登录 Splunk：打开浏览器，输入你的 Splunk 地址，输入管理员账号密码登录。
2. 进入 AOB 应用：在首页左侧应用列表中，找到并点击 Splunk Add-on Builder。
3. 打开项目：在列表中找到 PEAK-llm-analyzer 项目并点击进入。
4. 进入数据收集配置页：点击顶部导航栏的 Configure Data Collection。
5. 编辑任务模板：找到 PEAK AI Hunter 任务，点击右侧的 Edit 按钮。
6. 核对基础属性 (Tab 1)：在 Data Input Properties 标签页底部，确保 Collection interval (seconds) 是 60。
7. 改造参数表单 (Tab 2)：点击顶部的 Data Input Parameters 标签页。点击画布上那个无法删除的文本框，在右侧 Property Editor 面板修改：
   Internal Name 改为 custom_message
   Label 改为 Custom Heartbeat Message
   Default value 填入 Alive
8. 进入下一关：点击页面右上角的绿色 Next 按钮。

Step 2：注入心跳探针代码 (Define and Test 页面)

此时进入了 Define and Test 标签页。左侧是刚才配置的表单，右侧是代码编辑器。

详细操作路径：
1. 定位核心函数：在右侧编辑器向下滚动，找到 def collect_events(helper, ew): 这一行。
2. 清理默认代码：将这一行下方所有的默认代码全部删除。
3. 注入探针代码：将以下代码完整粘贴到该函数下方（注意保持 Python 缩进）：

def collect_events(helper, ew):
    import time
    import datetime
    try:
        # 1. 获取当前时间
        current_time = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        # 2. 获取被唤醒的任务实例名称
        stanza_name = helper.get_input_stanza_names()
        # 3. 获取目标索引
        target_index = helper.get_output_index() or "default_test_index"
        # 4. 读取自定义参数 (注意：数据输入必须使用 get_arg)
        custom_msg = helper.get_arg("custom_message")
        # 5. 构造日志内容
        log_msg = f"[PEAK AI Hunter Heartbeat] Time: {current_time} | Task: {stanza_name} | Target Index: {target_index} | Custom Msg: {custom_msg} | Status: Scheduler awakened the Python script."
        # 6. 将日志写入系统内部索引
        helper.log_info(log_msg)
    except Exception as e:
        helper.log_error(f"Heartbeat execution error: {str(e)}")

4. 执行语法测试：在左侧 Custom Heartbeat Message 框里输入 Test Run，点击右上角绿色的 Test 按钮。查看底部 Output 面板，只要没有红色报错即可。
5. 固化代码 (Save)：测试通过后，必须点击右上角绿色的 Save 按钮。看到保存成功提示后，关闭向导窗口。

Step 3：在前端创建任务，检索真实的心跳

代码保存后，需要去前端创建一个任务实例来激活调度器。

1. 点击左上角 Splunk 徽标返回首页。
2. 进入我们的应用：PEAK-llm-analyzer。
3. 点击顶部导航栏的 Inputs 标签页。
4. 点击右上角的 Create New Input，选择 PEAK AI Hunter。
5. 填入任务参数：
   Name: my_first_heartbeat
   Interval: 60
   Index: main
   Custom Heartbeat Message: Engine Started
6. 点击 Add 保存。此时，后台任务正式开始运转。

验证搜索日志：
等待一到两分钟。点击顶部的 Search 标签页，在搜索框输入：
index=_internal "[PEAK AI Hunter Heartbeat]"

将时间范围调整为 Last 15 minutes，点击搜索。你会看到类似于下面这样的日志，精准地每隔 60 秒出现一次：

INFO ExecProcessor - [PEAK AI Hunter Heartbeat] Time: 2026-03-26 21:46:24 | Task: my_first_heartbeat | Target Index: main | Custom Msg: Engine Started | Status: Scheduler awakened the Python script.

恭喜你！你现在已经彻底搞懂了 AOB 数据输入的底层原理。明天（Day 6），我们将在这个定时醒来的代码中加入 Splunk SDK，让它去抓取真实的业务日志！
