# 🚀 Day 5：任务调度器 —— 建立后台数据输入流与“心跳”机制 (Data Inputs)

**今日目标**：理解 Data Input（数据输入）的底层调度原理。我们将顺应 AOB 的强制参数规则配置输入界面，编写 Python 探针代码，并在前端真实启动任务。最终，我们要去 Splunk 的内部索引（`_internal`）中，亲眼见证代码被系统定时“唤醒”的运行轨迹。



### 🧠 极客白话：什么是 Data Input 与“心跳”机制？

对于初学者来说，Data Input（数据输入）听起来像是一个让你填表单的界面。但实际上，它是 Splunk 插件的**“心脏起搏器”**！

* **频率 (Interval) = 心率**：当你在界面上设置了 Interval 为 60 秒时，相当于给系统设定了一个定时闹钟。
* **执行 (Execution) = 心跳**：每隔 60 秒，Splunk 的底层调度器就会自动去唤醒并执行一次你写的 Python 脚本，就像心脏跳动一样。
* **多实例 (Multi-instance)**：你可以基于同一个 Python 脚本，在前端创建无数个任务（比如任务 A 每 60 秒查防火墙，任务 B 每 10 分钟查数据库），它们独立跳动，互不干扰。

今天，我们要给这颗心脏装上探针，记录下它的每一次跳动！

---

### 🛠️ Step 1：配置前端 UI (化被动为主动)

AOB 有一个硬性规定：模块化输入**必须至少保留一个自定义参数**，否则无法保存。我们不能把它删光，那就把它变成一个有用的字段。

1. 在浏览器中登录 Splunk (`http://localhost:8000`)。
2. 进入 **Splunk Add-on Builder** -> **PEAK-llm-analyzer** 项目。
3. 点击 **Configure Data Collection**，找到 `PEAK AI Hunter`，点击右侧的 **Edit (编辑)**。
4. 在第一页 **Data Input Properties** 中，确保底部的 **Collection interval** 设置为 `60` seconds（这是默认心率）。
5. 切换到 **Data Input Parameters** 标签页。
   * 你会看到画布上有一个无法删除的默认文本框。点击它，在右侧属性栏将其修改为：
   * **Internal Name**: `custom_message`
   * **Display Label**: `Custom Heartbeat Message`
   * **Default value**: `Alive`
6. 设置完毕后，点击页面右上角或下方的 **Next >** 按钮，正式进入代码定义与测试阶段。

---

### 💻 Step 2：注入“心跳探针”代码 (Define & Test)

现在我们来到了 **Define & Test** 标签页。在这里，左侧是你刚才定义的 `Custom Heartbeat Message`，右侧是代码编辑器。

**⚠️ 极客铁律**：代码注释与日志严格使用纯英文。

向下滚动找到 `def collect_events(helper, ew):`，将其下方的内容完全替换为以下代码：

```python
def collect_events(helper, ew):
    """Implement your data collection logic here"""
    import time
    import datetime
    
    try:
        # 1. Get the current execution time
        current_time = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        
        # 2. Get the specific instance name (Stanza) triggered by the scheduler
        stanza_name = helper.get_input_stanza_names()
        
        # 3. Get the target index configured by the user
        # Note: During the AOB "Test" click, this might return None. In real runs, it returns the Index.
        target_index = helper.get_output_index() or "default_test_index"
        
        # 4. Read the custom parameter we created in the UI
        custom_msg = helper.get_arg("custom_message")
        
        # 5. Construct the heartbeat log payload
        log_msg = f"[PEAK AI Hunter Heartbeat] Time: {current_time} | Task: {stanza_name} | Target Index: {target_index} | Custom Msg: {custom_msg} | Status: Scheduler awakened the Python script."
        
        # 6. Write this heartbeat log to Splunk's _internal index (splunkd.log)
        helper.log_info(log_msg)
        
    except Exception as e:
        helper.log_error(f"Heartbeat execution error: {str(e)}")
```

**模拟测试与保存**：
1. 在左侧的 `Custom Heartbeat Message` 框里随便填个词（比如 `Test Run`）。
2. 点击右上角的 **Test** 按钮。注意：这里的 Test 只是让代码单次空跑一下，测试语法有没有报错，此时你看不到 Interval 或 Index 的生效。
3. 只要底部的 Output 没有标红报错，直接点击 **Save**，然后关闭向导。

---

### 🔍 Step 3：在前端创建任务，检索真实的“心跳”

代码写好并保存后，我们需要像真实用户一样，去前端创建一个“任务实例”来激活底层的调度器。

1. 点击左上角的 `Splunk>enterprise` 徽标返回 Splunk 首页。
2. 在左侧进入我们的应用：**PEAK-llm-analyzer**。
3. 点击顶部导航栏的 **Inputs (输入)** 标签页。
4. 点击右上角的 **Create New Input** -> **PEAK AI Hunter**。
5. 填入真实的任务参数：
   * **Name**: `my_first_heartbeat`
   * **Interval**: `60`
   * **Index**: `main`
   * **Custom Heartbeat Message**: `Engine Started`
6. 点击 **Add** 保存。此时，你的任务已经脱离了手动测试，正式被系统调度器接管！

**🎉 见证奇迹的时刻（搜索日志）：**
等待一到两分钟，让调度器在后台跑完几轮。点击顶部的 **Search (搜索)** 标签页，在搜索框中输入：
```spl
index=_internal "[PEAK AI Hunter Heartbeat]"
```
将时间范围调整为 **Last 15 minutes**，点击搜索。

你会看到类似于下面这样的日志，**精准地每隔 60 秒出现一次**：
> `INFO ExecProcessor - ... [PEAK AI Hunter Heartbeat] Time: 2023-10-27 10:00:00 | Task: my_first_heartbeat | Target Index: main | Custom Msg: Engine Started | Status: Scheduler awakened the Python script.`
