# Day 1: Prepare Before Action —— Build a Splunk Development Environment from Scratch for Beginners
__Today's Goal__: Set up a clean, fully read-write accessible Splunk developer lab with unlocked enterprise-grade advanced features on Linux (WSL2) locally.

## Why Linux / WSL2 is a Must?

Splunk's Add-on Builder (AOB) and underlying file permission mechanisms (such as execution of `.sh` scripts, file path separators, etc.) are prone to counterintuitive bugs on Windows. Using **WSL2 (Windows Subsystem for Linux 2)** is currently the most elegant, high-performance, and production-environment-close local development solution.

---
## Step 1: Deep Installation and Verification of WSL2 and Ubuntu LTS
To ensure no weird bugs occur in future plugin packaging and Python environments, we must install the **latest Ubuntu Long-Term Support (LTS)** version with a pure WSL2 underlying architecture.
Open PowerShell on Windows **as an administrator** and follow the steps below strictly:
### 1.1 Current State Check

Enter the following command in PowerShell to check the current status of WSL and understand the baseline:

```powershell
wsl -l -v
```

* **Scenario A (Green Light)**: A prompt like `wsl is not recognized...` or connection timeout indicates your computer is a clean slate — perfect!
* **Scenario B (Yellow Light)**: Other distributions are listed. They can be retained if they do not affect work; if you want to start over, you can uninstall them with `wsl --unregister <Distribution Name>`.

View the list of available Linux distributions currently provided by the Microsoft Official Store:

```powershell
wsl --list --online
```

### 1.2 Install the Latest Ubuntu Long-Term Support (LTS) Version Specifically

Do not use the default `wsl --install` (it’s like opening a blind box); we need to accurately install the latest LTS version (assuming the latest is 24.04 at present):

```powershell
wsl --install -d Ubuntu-24.04
```

**Important Note**: If the command line prompts that a **restart is required** halfway through, be sure to restart your computer. After the restart, WSL will automatically pop up a terminal window to continue the installation.

At the end of the installation, the Ubuntu terminal will ask you to create a default UNIX account for the system:

* `Enter new UNIX username:` Enter a short English name (e.g., `hunter` or `admin`).
* `New password:` Enter a password (**Note: There is no asterisk prompt when entering the password in the Linux terminal; type it blindly and press Enter directly**).

### 1.3 In-Depth Verification of Installation Results

**First Verification: Confirm WSL Architecture Version (Execute in Windows PowerShell)**

```powershell
wsl -l -v
```

The **VERSION** column in the output **must be 2**! If it is 1, it means you have installed a crippled version (you can force an upgrade with `wsl --set-version Ubuntu-24.04 2`).

**Second Verification: Confirm Linux Internal Distribution (Execute in the configured Ubuntu terminal)**

```bash
cat /etc/os-release
```

The output should contain something like `PRETTY_NAME="Ubuntu 24.04 LTS"`.

## Step 2: Register a Developer Account and Obtain the "Immunity License"

Splunk's default Free trial version locks core features. To fully develop AI plugins, we must obtain an official **Developer License** for developers — 10GB/day with a validity period of six months — for free.

1. In the Windows browser, open the Splunk Developer official website: [dev.splunk.com](https://dev.splunk.com/)
2. Click **Sign Up** in the upper right corner, fill in your real email address to complete registration and log in.
3. Find **Enterprise** -> **Get a Developer License** in the top navigation bar (or look for Request License on your personal homepage).
4. Check to agree to the developer terms and click **Request License**.
5. Download the generated `splunk.license` XML file. **Save this file on your Windows desktop or Downloads folder for later use.**

---

## Step 3: Precisely Download Splunk Enterprise via Command Line

As geeks, we directly use `wget` in the Linux terminal to download the installation package at high speed.

1. Log in to the Splunk official website in the Windows browser and visit **Free Trials & Downloads** -> **Splunk Enterprise**.
2. Select **Linux** for the operating system and **.tgz** for the format.
3. After clicking Download, **immediately cancel the download in the Windows browser**. Find the **"Download via Command Line (wget)"** on the right side of the page and copy the complete command in the black box.
4. Return to your Ubuntu WSL terminal, update the software source and execute the download (replace the link in quotes with the real link you just copied):

```bash
# 1. Update software sources and ensure wget is installed
sudo apt update && sudo apt install wget -y

# 2. Paste the wget command you copied to pull the installation package
wget -O splunk-Linux-x86_64.tgz "https://download.splunk.com/products/splunk/releases/9.x.x/linux/splunk-9.x.x-xxxxxxx-Linux-x86_64.tgz"
```

---

## Step 4: Decompression and Directory Permission Configuration (Core Pitfall Avoidance!)

In the Linux world, the standard location for large third-party software is the `/opt` directory, but it is owned by `root` by default. **If you run Splunk with root privileges, you will not be able to modify the code generated by AOB later!** We must take over this directory properly:

1. **Use elevated privileges to decompress the file to the `/opt` directory**:

```bash
sudo tar -xzvf splunk-*.tgz -C /opt
```

2. **Transfer directory ownership (Extremely Important)**:
Transfer the ownership of the entire Splunk folder to the current ordinary UNIX account (`$USER` represents your current username):

```bash
sudo chown -R $USER:$USER /opt/splunk
```

*After executing this command, you are the absolute master of `/opt/splunk`!*

---

## Step 5: Start the Service and Inject the License

Everything is ready; start the service and unlock the feature restrictions.

### 5.1 First Startup and Account Initialization

```bash
# 1. Enter the directory of Splunk's executable files
cd /opt/splunk/bin

# 2. Start the service and automatically accept the license agreement
./splunk start --accept-license
```

The terminal will ask you to create a **local administrator account with the highest privileges**:

* Enter username: `admin`
* Enter password: `changeme` (type blindly, press Enter to confirm, need to enter twice).

When the terminal prints `The Splunk web interface is at http://<Computer Name or IP>:8000`, the startup is successful!

### 5.2 Log in to the Web Interface and Import the License

1. Open the Windows browser and visit: `http://localhost:8000` *(If you cannot access it, enter `ip a` in the Ubuntu terminal to find the IP of eth0 and access it with that IP)*.
2. Log in with `admin` and `changeme`.
3. In the black system navigation bar at the top of Splunk, click **Settings** -> **Licensing**.
4. Click the green **Add license** button.
5. Select **Choose file**, select the `splunk.license` file downloaded in Step 2 from your Windows desktop, and click **Install**.
6. Click the **Restart Splunk** button on the page to restart the system.

---

## Final Result Confirmation (Milestone Achieved Today)

After Splunk restarts and you log in again:

1. Enter **Settings** -> **Licensing** again.
2. Clearly see the words **Enterprise (Developer)** in the License group.
3. Confirm that you have an available indexing quota of **10,240 MB (10GB)** per day.

**Congratulations! The hardcore infrastructure setup for Day 1 is perfectly completed!**
Now you have an enterprise-grade development platform with perfectly configured permissions, fully unlocked core functions, and running on the native Linux kernel.
