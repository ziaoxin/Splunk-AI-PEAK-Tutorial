### 💻 Step 3: Write Your First AI Conversation Script

Now, we are going to write a Python script to say hello to Qwen. To make this absolutely beginner-friendly, we will use the simplest text editor, `nano`.

1. In the terminal, type the following command to create and open the file:
```bash
nano test_llm.py

```


2. **Copy and right-click to paste** the following code into the terminal:

```python
import requests
import json
import sys

# ==========================================
# 1. Base Configuration for Alibaba Cloud Qwen
# ==========================================
BASE_URL = "https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions"
MODEL_NAME = "qwen3.5-plus"
API_KEY = "sk-11f7c70b72bf4a3e81a6956d7917****"

# ==========================================
# 2. Prepare the Prompt (The 'messages' payload structure)
# ==========================================
messages = [
    {"role": "system", "content": "You are a senior cybersecurity expert. Speak extremely concisely."},
    {"role": "user", "content": "Explain what a SQL Injection attack is in one sentence."}
]

# ==========================================
# 3. Assemble and Send the Network Request
# ==========================================
headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

payload = {
    "model": MODEL_NAME,
    "messages": messages,
    "temperature": 0.1  # Lower values make the AI's response stricter and less random
}

print("🚀 Sending request to Alibaba Cloud LLM, please wait...\n")

try:
    # Send POST request, with a timeout of 60 seconds
    response = requests.post(BASE_URL, json=payload, headers=headers, timeout=60)
    res_json = response.json()
    
    # Print the raw, complete JSON response (to see its true form)
    print("=== 📦 Complete JSON Data Returned by LLM ===")
    print(json.dumps(res_json, indent=4, ensure_ascii=False))
    print("\n" + "="*40 + "\n")
    
    # Bulletproof Logic: If the LLM throws an error and returns no 'choices' field, intercept it early
    if "choices" not in res_json:
        print(f"❌ API Call Failed. Error Details:\n{res_json}")
        sys.exit(1)
        
    # Extract the actual answer content and the tokens consumed
    reply_content = res_json["choices"][0]["message"]["content"]
    total_tokens = res_json["usage"]["total_tokens"]
    
    print(f"🤖 AI Response:\n{reply_content}\n")
    print(f"💰 Total Tokens Consumed for this conversation: {total_tokens}")

except Exception as e:
    print(f"❌ A code-level exception occurred during the request: {e}")

```

### 🧠 Core Concepts Summary (Crucial for Debugging!)

As a beginner, you must stare at the JSON block above and understand the following two points. This is the lifeblood of how we will parse data in Splunk later:

1. **What exactly is the Input? (The `messages` list)**
LLMs have no memory. Every time you interact with it, you must send the entire conversation history packed as a list of dictionaries:
* `"role": "system"`: This is the opening statement used to set the LLM's persona (e.g., "You are a security expert").
* `"role": "user"`: Represents the current question asked by the human.


2. **How to extract the Output? (Parsing JSON Data)**
The LLM returns a massive chunk of data. We need to write code to peel it layer by layer like an onion:
* **Extracting the text answer**: You must peel back the dictionary layers -> `res_json["choices"][0]["message"]["content"]`.
* **Why write "Bulletproof" logic?** When an API runs out of funds, times out, or the model name is typed incorrectly, the JSON returned by the LLM will completely lack the `choices` array. Forcing extraction will cause Python to crash (`list index out of range`). The `if "choices" not in res_json` check in our code neutralizes this fatal crisis in advance!
* **Extracting the billing cost**: Peel the dictionary similarly -> `res_json["usage"]["total_tokens"]`.
