# 🚀 Day 4: Safety First — Configuring Global Credentials (Setup Parameters)

**Today's Objective**: Strictly follow the Add-on Builder (AOB) wizard rules to complete the initialization of the data collection task, design the global variable configuration page UI, enter real API sample data in the test wizard, and successfully save the final configuration.

---
### 🛠️ Step 1: Initialize the Data Input Base (Data Input Properties)

In AOB, the global settings (Setup) are attached to the "New Input" wizard. We must complete the basic data input flow first before we can unlock the setup page.

1. Log in to Splunk in your browser (`http://localhost:8000`).
2. Go to **Splunk Add-on Builder** -> click into your **PEAK-llm-analyzer** project.
3. Click **Configure Data Collection** on the top navigation bar.
4. Click the **New Input** button in the top right corner, and select the large icon: **Modular input using your own Python code**.

**🎯 Satisfy the Mandatory Rules**:
You will land on the **Data Input Properties** tab first. Strictly fill in the following information:
* **`*`Source type name**: `peak:llm:execution` *(Geek standard: Using colons indicates this is the LLM execution log under the PEAK framework)*
* **`*`Input display name**: `PEAK AI Hunter`
* **`*`Input name**: `peak_hunter_task` *(The underlying variable name; must be lowercase with underscores)*
* **Description**: `The automated threat hunting task driven by LLM.`
* **`*`Collection interval**: `300` seconds *(Crucial! The default is 30 seconds. We change it to 300 seconds/5 minutes to prevent bankrupting your API quota with frequent calls.)*

### 🛠️ Step 2: Build Your "Global Console" (Add-on Setup Parameters)

Move your eyes to the tab bar at the top of the wizard and click the **Add-on Setup Parameters** tab. This is where we design our all-English configuration interface.

**2.1 Create the Base URL Field**
1. Drag and drop the **Text** component from the left panel onto the center canvas.
2. Fill in the properties on the right:
   * **Internal Name**: `base_url`
   * **Label**: `LLM Base URL`
   * **Help string**: `Enter an OpenAI-compatible API endpoint URL.`

**2.2 Create the Model Name Field**
1. Drag and drop another **Text** component onto the canvas.
2. Fill in the properties on the right:
   * **Internal Name**: `model_name`
   * **Label**: `LLM Model Name`
   * **Help string**: `Enter the exact model name (e.g., qwen3.5-plus, gpt-4o, deepseek-chat).`

**2.3 Create the API Key Field (🛡️ Top Secret Security!)**
1. Drag and drop a **Password** component from the left to the bottom of the canvas.
2. Fill in the properties on the right:
   * **Internal Name**: `api_key`
   * **Label**: `API Key`
   * **Help string**: `Enter your API key (sk-...). This credential will be securely encrypted by Splunk's credential store.`

### 🧠 Geek Deep Dive: Where does Splunk hide your passwords?

When you click Save, Splunk writes the plain text URL and Model Name into `/opt/splunk/etc/apps/TA-peak-llm-analyzer/local/app.conf`. However, even if you dig through all the text files, you will **never find** that API Key string! It has been encrypted and stored in the underlying `passwords.conf` file.

In the Python code we will write in the future, we only need one highly elegant official built-in function:
```python
api_key = helper.get_global_setting("api_key")
```
Splunk will automatically decrypt it in memory and feed it to our code, ensuring ultimate security!
