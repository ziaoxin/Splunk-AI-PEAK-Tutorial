# 🚀 Day 5: Task Scheduler — Establish Data Inputs and the "Heartbeat" Mechanism

**Today's Objective**: Understand the underlying scheduling principles of Data Inputs. We will conform to AOB's mandatory parameter rules to configure the input UI, write a Python probe script, and officially launch the task in the frontend. Finally, we will search Splunk's internal index (`_internal`) to witness the code being automatically "awakened" by the system on a schedule.

---

### 🧠 Geek Talk: What is a Data Input and the "Heartbeat" Mechanism?

To a beginner, a Data Input might sound like just a form-filling interface. In reality, it is the **"Pacemaker"** of your Splunk Add-on!

* **Interval = Heart Rate**: When you set an Interval of 60 seconds, you are setting a recurring alarm for the system.
* **Execution = Heartbeat**: Every 60 seconds, Splunk's underlying scheduler automatically wakes up and executes your Python script, just like a heartbeat.
* **Multi-instance**: You can create countless tasks based on the same Python script (e.g., Task A checks the firewall every 60s, Task B checks the database every 10 mins). They beat independently without interfering with each other.

Today, we are going to attach a probe to this heart to record its every beat!

---

### 🛠️ Step 1: Configure the Frontend UI (Turn Constraint into Advantage)

AOB has a strict rule: a modular input **must retain at least one custom parameter** to be saved. Since we can't delete everything, let's turn the default text box into a useful field.

1. Open Splunk in your browser (`http://localhost:8000`).
2. Go to **Splunk Add-on Builder** -> **PEAK-llm-analyzer** project.
3. Click **Configure Data Collection**, find `PEAK AI Hunter`, and click **Edit**.
4. On the first page, **Data Input Properties**, ensure the **Collection interval** at the bottom is set to `60` seconds (the default heart rate).
5. Switch to the **Data Input Parameters** tab.
   * You will see an undeletable default text box on the canvas. Click it, and modify its properties on the right:
   * **Internal Name**: `custom_message`
   * **Label**: `Custom Heartbeat Message`
   * **Default value**: `Alive`
6. Once configured, click the **Next >** button to officially enter the code definition and testing phase.

---

### 💻 Step 2: Inject the "Heartbeat Probe" Code (Define & Test)

We are now on the **Define & Test** tab. The left side shows the `Custom Heartbeat Message` you just defined, and the right side is the Code Editor.

**⚠️ Geek Iron Rule**: Comments and logs strictly use pure English.

Scroll down to locate `def collect_events(helper, ew):`. Completely replace the content below it with the following code:

```python
def collect_events(helper, ew):
    """Implement your data collection logic here"""
    import time
    import datetime
    
    try:
        # 1. Get the current execution time
        current_time = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        
        # 2. Get the specific instance name (Stanza) triggered by the scheduler
        stanza_name = helper.get_input_stanza_names()
        
        # 3. Get the target index configured by the user
        # Note: During the AOB "Test" click, this might return None. In real runs, it returns the Index.
        target_index = helper.get_output_index() or "default_test_index"
        
        # 4. Read the custom parameter we created in the UI
        custom_msg = helper.get_arg("custom_message")
        
        # 5. Construct the heartbeat log payload
        log_msg = f"[PEAK AI Hunter Heartbeat] Time: {current_time} | Task: {stanza_name} | Target Index: {target_index} | Custom Msg: {custom_msg} | Status: Scheduler awakened the Python script."
        
        # 6. Write this heartbeat log to Splunk's _internal index (splunkd.log)
        helper.log_info(log_msg)
        
    except Exception as e:
        helper.log_error(f"Heartbeat execution error: {str(e)}")
```

**Simulation Test and Save**:
1. Type a random phrase (e.g., `Test Run`) into the `Custom Heartbeat Message` box on the left.
2. Click the **Test** button in the top right. *Note: This merely executes your code once synchronously to check for syntax errors. You will not see the Interval or Index logic apply here.*
3. As long as there are no red error messages in the Output panel, click **Save**, then close the wizard.

---

### 🔍 Step 3: Create a Task and Retrieve the True "Heartbeat"

With the code saved, we need to act as an end-user and create a "task instance" in the frontend to activate the underlying scheduler.

1. Click the `Splunk>enterprise` logo in the top left to return to the Splunk home page.
2. Enter our app: **PEAK-llm-analyzer**.
3. Click the **Inputs** tab on the top navigation bar.
4. Click **Create New Input** in the top right -> select **PEAK AI Hunter**.
5. Fill in the real task parameters:
   * **Name**: `my_first_heartbeat`
   * **Interval**: `60`
   * **Index**: `main`
   * **Custom Heartbeat Message**: `Engine Started`
6. Click **Add** to save. Your task has now graduated from manual testing and is officially taken over by the system scheduler!

**🎉 The Moment of Magic (Searching the Logs):**
Wait a minute or two to let the scheduler run a few background cycles. Click the **Search** tab at the top, and enter the following in the search box:
```spl
index=_internal "[PEAK AI Hunter Heartbeat]"
```
Adjust the time range to **Last 15 minutes** and click search.

You will see logs exactly like this, appearing **punctually every 60 seconds**:
> `INFO ExecProcessor - ... [PEAK AI Hunter Heartbeat] Time: 2023-10-27 10:00:00 | Task: my_first_heartbeat | Target Index: main | Custom Msg: Engine Started | Status: Scheduler awakened the Python script.`

**Congratulations!** You now completely understand the underlying mechanics of AOB Data Inputs. Tomorrow (Day 6), we will integrate the Splunk SDK into this periodically awakening script to fetch real business logs!
