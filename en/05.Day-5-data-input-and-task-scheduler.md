### 🧠 Geek Talk: What is a Data Input and the "Heartbeat" Mechanism?

To a beginner, a Data Input might sound like a simple form-filling interface. But to a geek, it is the **"Pacemaker"** of your Splunk Add-on!

* **Interval = Heart Rate**: The 60-second interval you set in the UI is essentially setting a "recurring wake-up alarm" for the system.
* **Execution = Heartbeat**: Every 60 seconds, Splunk's underlying scheduler (`ExecProcessor`) automatically wakes up and executes your Python script (the `collect_events` function), just like a heartbeat.
* **The Significance of a Probe**: Before we start pulling massive amounts of real data, we must first inject a "probe" into our script to verify that this awakening mechanism is actually alive.

Today, we are going to inject the first surge of power into our AI engine: making it open its eyes on schedule!

### 🛠️ Step 1: Configure the Frontend UI (Turn Constraint into Advantage)

AOB has a strict system rule: **A modular input must retain at least one custom parameter**, otherwise it cannot be saved to proceed. Since we can't delete everything, let's transform it into a useful "probe field."

1. **Enter AOB Project**: From the Splunk home page, enter **Splunk Add-on Builder** and open our **PEAK-llm-analyzer** project.
2. **Locate Configuration Page**: Click **Configure Data Collection** in the top navigation bar.
3. **Edit Task Template**: Find the `PEAK AI Hunter` created on Day 4 and click **Edit** on the right to enter the wizard.
4. **Transform Parameters (Tab 2)**: Switch to the **Data Input Parameters** tab.
   * Click on the undeletable "Text Input" on the canvas.
   * Modify it in the right-hand Property panel: change **Internal Name** to `custom_message`; change **Display Label** to `Custom Heartbeat Message`; change **Default value** to `Alive`.
5. **Proceed to Next Level**: Click the green **Next >** button in the top right corner.

---

### 💻 Step 2: Inject the "Heartbeat Probe" Code (Define & Test Page)

You are now in the code laboratory. The editor on the right is our main battlefield.

1. **Clear Default Code**: Locate `def collect_events(helper, ew):` and delete all placeholder code beneath it.
2. **Inject Probe Code**: Paste the following logic-corrected code.
   * **⚠️ Geek Pitfall Guide**: You MUST use `helper.get_arg` instead of `get_param` to read Data Input variables, otherwise the code will silently crash!

```python
def collect_events(helper, ew):
    """Implement your data collection logic here"""
    import time
    import datetime
    
    try:
        # 1. Get the current execution time (The moment of heartbeat)
        current_time = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        
        # 2. Get the specific instance name (Stanza)
        stanza_name = helper.get_input_stanza_names()
        
        # 3. Get the target index (Where data will be queried/stored)
        target_index = helper.get_output_index() or "default_test_index"
        
        # 4. Read the custom parameter (MUST use get_arg for Data Inputs!)
        custom_msg = helper.get_arg("custom_message")
        
        # 5. Construct the heartbeat log payload
        log_msg = f"[PEAK AI Hunter Heartbeat] Time: {current_time} | Task: {stanza_name} | Target Index: {target_index} | Custom Msg: {custom_msg} | Status: Scheduler awakened the Python script."
        
        # 6. Write to Splunk's internal logs for verification
        helper.log_info(log_msg)
        
    except Exception as e:
        helper.log_error(f"Heartbeat execution error: {str(e)}")
```

3. **Crash-Proof Test (Test)**: Type a random word into the input box on the left, and click the green **Test** button in the top right.
   * **Geek Experience**: If it shows a green **Done** and no errors, it means the syntax is correct. Even though you won't see the log output here (because it's written directly to the underlying file), it confirms the logic runs successfully.
4. **Commit Code (Save)**: You **MUST** click the **Save** button in the top right corner.

### 🔍 Step 3: Create a Task in the Frontend, Retrieve the True "Heartbeat"

Code is just a "blueprint"; now we need to recruit a "physical worker" to officially launch it.

1. **Return to App**: Go back to the Splunk home page and enter our app **PEAK-llm-analyzer**.
2. **Create Input**: Click the **Inputs** tab -> **Create New Input** -> **PEAK AI Hunter**.
3. **Fill in Parameters**:
   * **Name**: `my_first_heartbeat`
   * **Interval**: `60`
   * **Index**: `main`
   * **Custom Heartbeat Message**: `Engine Started`
4. **Save and Activate**: Click **Add**. The background scheduler has now taken over your code.
