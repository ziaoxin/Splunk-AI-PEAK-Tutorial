### 🧠 Geek Talk: Why M-ATH Clustering is Essential for "Unknown" Logs

When dealing with massive, unstructured logs, simple random sampling is highly inefficient as it likely captures redundant background noise.

* **The Power of M-ATH**: In the PEAK framework, M-ATH is the most effective approach for "Cold Start" tasks. Using Splunk's native `cluster` algorithm, the system automatically groups similar logs and calculates the frequency of each pattern.
* **Finding the "Islands"**: True security threats often hide in "island" logs—patterns that appear with extremely low frequency (e.g., 1 or 2 occurrences out of 50,000 events). Today, our Python script stops "blindly" grabbing logs and starts using algorithms to identify these rare patterns for the AI to analyze.

```python
def collect_events(helper, ew):
    """Implement M-ATH (Clustering) logic to extract rare log patterns"""
    
    helper.log_info("PEAK AI Hunter started M-ATH data collection cycle.")

    try:
        # 1. Robustly acquire the session_key from the context
        session_key = getattr(helper, 'session_key', None)
        
        # Fallback: extract from internal input_definition metadata
        if not session_key and hasattr(helper, '_input_definition'):
            session_key = getattr(helper._input_definition, 'metadata', {}).get('session_key')
            
        if not session_key:
            helper.log_error("Failed to acquire session_key. Authentication failed.")
            return

        # 2. Connect to the Splunk service
        service = client.Service(token=session_key)

        # 3. Get the user-configured Index
        target_index = helper.get_output_index() or "main"

        # 4. Construct the M-ATH Clustering SPL
        # Geek Tip: We use -24h to provide a robust statistical baseline for identifying 
        # "rare" patterns across daily cycles, while `head 50000` acts as a 
        # performance safety valve to ensure fast execution.
        search_query = (
            f"search index={target_index} earliest=-24h latest=now "
            f"| head 50000 "
            f"| cluster showcount=t "
            f"| sort count "
            f"| head 5 "
            f"| table cluster_count, _raw"
        )
        
        # 5. Execute the oneshot search
        kwargs_oneshot = {"output_mode": "json"}
        helper.log_info(f"Executing M-ATH Clustering SPL: {search_query}")
        
        oneshot_search_results = service.jobs.oneshot(search_query, **kwargs_oneshot)

        # 6. Parse results
        reader = results.JSONResultsReader(oneshot_search_results)
        
        rare_patterns = []
        for result in reader:
            if isinstance(result, dict) and "_raw" in result:
                # Store the rare pattern along with its occurrence count
                rare_patterns.append(f"Count: {result.get('cluster_count')} | Raw: {result['_raw']}")

        # 7. Verification log
        if len(rare_patterns) > 0:
            helper.log_info(f"Successfully identified {len(rare_patterns)} rare log patterns via M-ATH.")
            helper.log_info(f"Sample Rare Pattern [1]: {rare_patterns[0][:200]}...")
        else:
            helper.log_warning(f"M-ATH executed but found 0 events in index '{target_index}'. Check your data.")

    except Exception as e:
        helper.log_error(f"Execution error during M-ATH search: {str(e)}")
```
